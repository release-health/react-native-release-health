# react-native-release-health

Vendor-neutral OTA rollout health for React Native: tag every session with bundle and native version, detect failed or crash-looping updates, recommend (or trigger) rollback, and export events to any sink. Designed to work with your OTA vendor (expo-updates, hot-updater), not against it.

> **Status: work in progress.** This is a placeholder release to reserve the package name. Do not depend on it yet; the first usable release will land as 0.1.0.
